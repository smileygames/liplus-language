# Repository Requirements: Li  Enabled の条件

このページでは、  
**どのようなリポジトリが Li+ に対応していると見なされるか**  
その最小要件を定義します。

Li+ はツールやプラットフォームを指定しません。  
必要なのは「構造」であり、「実装手段」ではありません。

---

## Definition

次の条件を満たすリポジトリを、  
**Li+ Enabled Repository** と呼びます。

---

## Required Components

### 1. Li+.md

リポジトリには、  
**Li+ の判断命令文である `Li+.md`** が存在しなければなりません。

- AI が従うべき意味論・制約が記述されていること
- プロジェクト固有の Li+ 実装であってよい
- このファイルは **AI 向け憲法**として扱われる

---

### 2. Issue → Execution Link

すべての変更は、  
**明示的な Issue（SPEC）から始まる**必要があります。

- Issue は仮説・意図・制約を含む
- 実装・実行は Issue と紐づいて追跡可能であること
- Issue を持たない変更は、Li+ において無効とされます

---

### 3. Executable Process

リポジトリには、  
**実行可能なプロセス**が存在しなければなりません。

- CI
- スクリプト
- 手動実行手順

形式は問いませんが、次を満たす必要があります。

- 実装が実際に実行される
- 実行結果が再現可能である
- 実行が証拠を生む

---

### 4. Persistent Execution Evidence

実行結果は、  
**永続的な証拠として保存**されなければなりません。

含まれるもの：

- logs
- exit codes
- diffs
- artifacts

証拠は：

- 後から参照できる
- Issue や実装と対応づけられる
- 実行された「事実」として扱われる

---

### 5. Traceable History

リポジトリには、  
**仮説から実行までの履歴**が残っている必要があります。

- Issue（仮説）
- Commit / PR（実装）
- CI logs / artifacts（実行証拠）

重要なのは：

> **なぜその変更が行われ、  
> どの実行で何が起きたかが辿れること**

履歴の美しさは要求されません。

---

## Optional Components

以下は必須ではありませんが、  
Li+ を運用しやすくします。

- GitHub Actions や類似 CI
- 自動生成された build tags
- PR テンプレート
- Issue テンプレート
- Wiki（人間向け説明）

---

## Platform Independence

Li+ は特定のプラットフォームに依存しません。

- GitHub
- GitLab
- ローカルリポジトリ
- 社内 CI

いずれでも構いません。

重要なのは：

> **実行され、証拠が残ること**

---

## What Does NOT Matter

次の要素は、Li+ Enabled の条件ではありません。

- 使用しているプログラミング言語
- フレームワーク
- コードの美しさ
- 自動デプロイの有無
- 完全なテスト網羅率

Li+ は品質基準ではありません。

---

## Why These Requirements Exist

これらの要件は、  
AI と人間が **同じ現実を参照するため**に存在します。

- 推論は揮発する
- 会話は消える
- 記憶は歪む

だから Li+ は、

> **履歴と証拠を、リポジトリに固定する**

という選択をします。

---

## Summary

Li+ Enabled Repository とは：

- 判断命令（Li+.md）を持ち
- 仮説（Issue）を明示し
- 実行し
- 証拠を残し
- 履歴を辿れる

リポジトリです。

Li+ は、  
**構造が整った場所でのみ機能します。**
